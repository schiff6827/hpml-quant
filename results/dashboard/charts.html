Scaling vs. Precision: Qwen2.5 Results Dashboard

Static export of the 96 GB VRAM scale/precision experiment. The full runnable grid contains 15 cells per primary sweep: Qwen2.5-Instruct 7B, 14B, 32B, and 72B across unquantized 16-bit, GPTQ-Int8, AWQ-Int4, and BnB-NF4, excluding unquantized 72B because it does not fit in memory.

Charts Run List
Primary paired sweeps are ShareGPT performance with GSM8K quality, and random 512/256 performance with MMLU quality. System metrics are recorded for the full grid; Nsight Systems charts are representative kernel-level profiling slices.

ShareGPT + GSM8K

ShareGPT performance uses 500 prompts, infinite request rate, and max concurrency 64. GSM8K quality uses 8-shot prompting with a 250-example limit and strict exact-match accuracy.

Random 512/256 + MMLU

Random performance uses 500 requests with 512 input tokens and 256 output tokens. MMLU quality uses 5-shot prompting across all 57 subtasks with a limit of 5 examples per subtask.

Context Capacity

Context-capacity runs are separate from the fixed 10 GiB KV-cache performance sweeps. These runs use GPU memory utilization 0.95 and read vLLM's reported GPU KV-cache token capacity.

Nsight Systems Profiling

Selected representative runs were profiled with Nsight Systems to explain hardware behavior. These figures summarize kernel categories and achieved compute utilization.
